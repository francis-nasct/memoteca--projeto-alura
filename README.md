<p>
  Memoteca é um projeto para colecionar memórias e pensamentos. Ele foi desenvolvido durante o curso de JavaScript da Alura com foco na manipulação de dados com operações CRUD (Create, Read, Update e Delete).

  Foi disponibilizado um arquivo contendo a base do HTML e do CSS para que o foco fosse o desenvolvimento do JavaScript. A pós os preparativos iniciais, aprendi como utilizar o JSON-Server para simular um back-end, como fazer uma requisição à API com Fetch, e como renderizar dados na interface dinamicamente.
  
  Explorando a comunicação cliente-servidor, fiz o envio de dados usando uma requisição HTTP POST. Configurei o método da requisição como POST, defini o cabeçalho Content-Type como application/json e enviei os dados no corpo da requisição em formato JSON.
  
  Atualizei dinamicamente a interface para exibir dados (os pensamentos no mural); aprendi a manipular eventos de formulário, adicionando event listeners para tratar a submissão do formulário e o cancelamento da operação; implementei uma função para limpar os campos após a submissão ou cancelamento, preparando-o para novas entradas.
  
  Utilizei o método HTTP PUT para alterar dados no servidor; busquei um pensamento específico por ID através de uma requisição HTTP GET; preenchi o formulário de edição com dados existentes para facilitar a atualização; adicionei dinamicamente botões de edição à interface para cada item da lista de pensamentos; integrei as funções de busca e edição de pensamentos no código principal da aplicação; entendi como tratar erros nas requisições.
  
  Utilizei o método HTTP DELETE para remover dados do servidor; busquei um pensamento específico por ID para exclusão utilizando uma requisição; implementei uma função de exclusão para enviar a requisição de remoção à API; adicionei dinamicamente botões de exclusão à interface para cada item da lista de pensamentos; manipulei eventos de clique no botão de exclusão para atualizar a interface após a remoção de dados na API.
  
  Por fim, com este projeto também compreendi a utilização do Axios como alternativa ao fetch, e a facilidade de inclusão da biblioteca via CDN, permitindo fazer requisições GET, POST, PUT e DELETE de forma mais concisa.
</p>
